# Техническое задание: DevAgent Hub

## 1. Концепция

DevAgent Hub - собственный веб-интерфейс для управления AI-агентами и рабочими окружениями разработки.
OpenHands UI больше не является основным интерфейсом. OpenHands может временно оставаться в репозитории как исходный reference/runtime-компонент, пока backend полностью не будет заменен собственными сервисами.

Ключевая цель: пользователь клонирует репозиторий, запускает terminal-first установщик, выбирает модели, после чего получает готовый веб-интерфейс с агентами, редактором кода, терминалом, GitHub-автоматизацией и настройками.

## 2. Основной веб-интерфейс

Интерфейс строится с нуля в `apps/web` и должен работать как рабочая среда, а не как лендинг.

Обязательные разделы:

- **Chat** - постановка задачи агентам, запуск цепочки, отображение результата.
- **Agents** - управление агентами, порядок выполнения, включение/выключение, системные промпты и модель каждого агента.
- **Code** - встроенный OpenVSCode Server через iframe/прокси; это основной редактор кода вместо самописного Monaco-only редактора.
- **Terminal** - терминал рабочего окружения. На первом этапе допускается использовать терминал OpenVSCode Server; отдельный xterm.js-канал подключается backend-ом позже.
- **Preview** - встроенный просмотр локального веб-приложения или сервиса.
- **GitHub** - авторизация, создание репозитория, commit, push, pull request, просмотр состояния git.
- **Logs** - потоковые логи выполнения агентов, системных команд и интеграций.
- **Settings** - выбор языка RU/EN, настройки моделей по назначению, runner mode, URL OpenVSCode Server, preview URL, GitHub-права.

Дополнительные функции:

- переключатель русского и английского языка;
- назначение моделей по задачам: планирование, написание кода, ревью, тестирование, финальный ответ;
- статус интеграций: agent API, OpenVSCode Server, GitHub, terminal transport;
- настройки должны сохраняться локально и синхронизироваться с backend-конфигом там, где это безопасно.

## 3. Модели и агенты

Выбор моделей происходит на этапе установщика. Только выбранные модели записываются в `configs/agents.json` и отображаются в веб-интерфейсе.

Базовая цепочка агентов:

- **Generator** - планирование и первичная генерация решения.
- **Critic** - ревью архитектуры, рисков и качества.
- **Optimizer** - улучшение кода и производительности.
- **Tester** - тесты, проверки, smoke-сценарии.
- **Finalizer** - финальная сборка ответа и отчета.

Настройки моделей:

- модель можно назначить каждому агенту вручную;
- в Settings можно назначить модели по назначению;
- интерфейс применяет назначение по назначению к агентам автоматически;
- cloud-модели требуют ключи API, локальные модели работают через Ollama или совместимый backend.

## 4. Backend

Основной backend - FastAPI-сервис в `services/agent-api`.

Минимальные endpoints:

- `GET /health` - состояние сервиса.
- `GET /api/agents/config` - текущая конфигурация агентов и моделей.
- `POST /api/agents/config` - сохранение конфигурации.
- `POST /api/agents/run` - запуск цепочки агентов.
- `GET /api/agents/status/{task_id}` - состояние задачи.
- `GET /api/agents/logs/{task_id}` - поток логов через SSE/WebSocket.

Следующие backend-модули должны быть добавлены:

- менеджер OpenVSCode Server: запуск, остановка, проверка health, выдача URL;
- terminal gateway: xterm.js/WebSocket к sandbox shell;
- GitHub service: OAuth/token, создание репозитория, commit, push, PR;
- workspace service: выбор рабочей папки, безопасный доступ к файлам, ограничения на команды;
- secrets service: хранение API keys и GitHub token без записи секретов в git.

## 5. OpenVSCode Server

OpenVSCode Server используется как готовый редактор кода уровня VS Code.

Требования:

- установщик должен уметь скачивать и запускать OpenVSCode Server;
- веб-интерфейс должен открывать его во вкладке Code;
- путь workspace должен совпадать с рабочим каталогом агентов;
- терминал внутри OpenVSCode Server должен быть доступен пользователю;
- позже backend должен управлять lifecycle процесса, портом и авторизацией.

## 6. GitHub

GitHub-интеграция должна позволять агенту работать с репозиториями от имени пользователя, но с явным контролем.

Обязательные операции:

- подключить аккаунт через OAuth или token;
- создать новый репозиторий;
- инициализировать git в workspace;
- сделать commit;
- выполнить push;
- создать pull request;
- показать diff перед опасными действиями;
- запросить подтверждение для destructive-операций.

## 7. Установщик

Основной установщик - terminal-first CLI, без обязательного Electron.

Сценарий для Ubuntu Server:

1. Клонирование репозитория.
2. Запуск `install.sh` или Node CLI.
3. Проверка и установка зависимостей: Python 3.12+, Node.js 22+, Docker/Ollama при необходимости.
4. Выбор моделей: local/cloud/custom.
5. Запись выбранных моделей в `configs/agents.json`.
6. Установка frontend/backend зависимостей.
7. Установка и настройка OpenVSCode Server.
8. Настройка systemd-сервисов.
9. Запуск DevAgent Hub и вывод URL.

Windows-сценарий должен поддерживать PowerShell/winget и запуск локального dev/prod окружения.

Electron допускается только как optional desktop launcher, не как основной серверный установщик.

## 8. Локализация

Интерфейс должен поддерживать русский и английский язык.

Минимум:

- все основные заголовки, кнопки и статусы переводятся;
- язык можно переключить в верхней панели или Settings;
- выбранный язык сохраняется локально.

## 9. Нефункциональные требования

- UI должен быть плотным, рабочим и похожим на IDE/dashboard, без маркетингового лендинга.
- Долгие задачи агентов не должны блокировать интерфейс.
- Логи должны обновляться потоково.
- API keys и токены нельзя хранить в открытом виде в репозитории.
- Команды агентов должны выполняться в ограниченном workspace.
- Для России установщик должен поддерживать proxy/mirrors для npm, pip, Docker, GitHub и API-провайдеров.

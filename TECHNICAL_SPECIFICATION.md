# Полное техническое задание: AI Agent Studio (форк OpenHands)

## 1. Общая концепция

Создать **кастомный веб-интерфейс для управления AI-агентами**, основанный на кодовой базе OpenHands, со следующими ключевыми отличиями:

- **Мультиагентность** — цепочка агентов (Generator, Critic, Optimizer, Tester, Finalizer), которыми можно управлять через интерфейс.
- **Гибкая настройка моделей** — для каждого агента отдельно выбирается модель (локальная или облачная).
- **Установщик** — графический (Electron) для Linux и Windows, автоматизирующий установку, настройку моделей и запуск.
- **Безопасность** — учёт блокировок в РФ: рекомендация VPN/прокси, использование зеркал, настройка прокси через интерфейс.

## 2. Функциональные требования

### 2.1. Веб-интерфейс (форк OpenHands)

**Базовый функционал (как в OpenHands):**

- **Чат** — общение с агентами, отправка задач, отображение ответов с подсветкой кода.
- **Терминал** — интерактивный терминал (xterm.js) для выполнения команд.
- **Редактор кода** — Monaco Editor с поддержкой синтаксиса, автодополнения, подсветки.
- **Файловый менеджер** — навигация по файловой системе проекта, создание/удаление/переименование файлов и папок.
- **Браузер** — встроенный просмотр веб-страниц (iframe с ограничениями).
- **GitHub-интеграция** — авторизация через OAuth, просмотр репозиториев, создание коммитов, push, создание Pull Request.

**Новый функционал (добавляем):**

- **Панель управления агентами** — список всех доступных агентов с чекбоксами включения/выключения.
- **Редактирование агентов**:

- Добавление нового агента через UI (имя, системный промпт, выбор модели).
- Удаление агента.
- Порядок агентов (перетаскивание для изменения последовательности).
- **Настройка модели для каждого агента** — выпадающий список с доступными локальными и облачными моделями.
- **Отображение «мышления» агента** — в реальном времени показывать пошаговые логи (какие запросы отправляются, что возвращается, промежуточные выводы).
- **Статус выполнения** — визуальный индикатор (progress bar) для долгих задач.

### 2.2. Бэкенд (FastAPI + AutoGen)

**Основная логика:**

- Используем существующий `agent_system.py` (мультиагентная система на AutoGen) как отдельный сервис.
- OpenHands будет проксировать запросы к этому сервису, либо мы полностью заменим встроенного агента OpenHands на нашу систему.
- Все настройки моделей и агентов хранятся в конфигурационном файле (`agents.yaml` или `agents.json`), который синхронизируется с UI.

**Эндпоинты:**

- `POST /api/agents/run` — запуск цепочки агентов с переданными параметрами (задача, список агентов, модели для каждого).
- `GET /api/agents/status/{task_id}` — получение статуса выполнения.
- `GET /api/agents/logs/{task_id}` — получение пошаговых логов в реальном времени (через WebSocket или Server-Sent Events).
- `POST /api/agents/config` — сохранение конфигурации агентов и моделей.

### 2.3. Установщик (GUI на Electron)

**Общие требования:**

- Кросс-платформенность: **Linux (Ubuntu/Debian)** и **Windows**.
- Установка должна быть максимально автоматизирована, с минимальным вводом от пользователя.

**Шаги установщика:**

1. **Приветствие** — описание системы, ссылка на документацию.
1. **Проверка системы**:

- Наличие Docker, Python 3.10+, Node.js (для фронтенда).
- Если чего-то нет — предложить автоматически установить (для Linux через apt, для Windows через winget/choco или скачать установщики).
1. **Выбор модели**:

- **Локальная модель** (через Ollama):

- Список моделей с описанием:

- `qwen2.5-coder:7b` — хороший баланс скорости и качества для тестов.
- `deepseek-coder:6.7b` — сильная модель для написания кода.
- `deepseek-coder:33b` — максимальное качество, но требует 16+ ГБ ОЗУ.
- `llama3.2:3b` — очень быстрая модель для простых задач.
- Для каждой модели указать требования к ОЗУ и диску.
- **Облачная модель** (DeepSeek, OpenAI, OpenRouter, Custom):

- Поля: `API Key`, `Base URL`, `Model Name`.
- **Без модели** — если пользователь хочет только использовать OpenHands без мультиагентности.
1. **Настройка VPN/прокси** (опционально):

- Простое предложение: «В России могут быть блокировки. Рекомендуем использовать VPN. Если у вас есть прокси, укажите его в настройках после установки».
- Возможность сразу ввести адрес прокси (HTTP/SOCKS) для использования во всех сервисах.
1. **Путь установки** — выбор каталога (по умолчанию `~/ai-agent-studio`).
1. **Сборка и установка**:

- Клонирование форка OpenHands.
- Сборка фронтенда (`npm install && npm run build`).
- Сборка бэкенда (установка Python зависимостей).
- Генерация systemd-сервисов (Linux) или служб Windows.
- Запуск всех сервисов.
1. **Завершение** — поздравление, ссылка для открытия веб-интерфейса.

### 2.4. Учёт блокировок в России

- **Docker** — использовать официальные зеркала (например, `mirror.gcr.io`).
- **pip** — использовать зеркало Яндекса или `https://pypi.org/simple/` с `--index-url`.
- **npm** — использовать зеркало (например, `https://registry.npmmirror.com`).
- **GitHub** — через прокси или VPN, если он заблокирован. В установщике предложить настроить прокси.
- **API DeepSeek / OpenAI** — если заблокированы, предложить использовать OpenRouter или другие прокси-сервисы.

## 3. Техническая архитектура

### 3.1. Модификации OpenHands

**Фронтенд (React):**

- Добавить компоненты в `frontend/src/components/`:

- `AgentPanel/` — панель управления агентами.
- `AgentConfig/` — модальное окно для редактирования агента.
- `ThinkingDisplay/` — компонент для отображения логов.
- Изменить `frontend/src/components/chat/Chat.tsx` — добавить отправку задачи на мультиагентный бэкенд.
- В `frontend/src/components/settings/Settings.tsx` добавить вкладку «Агенты» с настройками моделей.

**Бэкенд (Python):**

- Оставить основной бэкенд OpenHands, но добавить новый маршрут `/api/agents` для работы с мультиагентной системой.
- Вместо встроенного агента OpenHands использовать вызов вашего `agent_system.py` через subprocess или HTTP-запрос к запущенному сервису на порту 8000.

**Альтернатива:** полностью заменить агента OpenHands на ваш `agent_system.py`, убрав лишнюю логику.

### 3.2. Установщик (Electron)

**Структура:**

```
installer/
├── main.js
```

**Логика установки:**

1. Проверка наличия Docker → если нет, предложить установить.
1. Проверка Python 3.10+ → если нет, предложить установить.
1. Проверка Node.js 18+ → если нет, предложить установить.
1. Клонирование форка OpenHands (с вашими изменениями) в выбранную папку.
1. Установка Python-зависимостей (venv + pip).
1. Установка Node-зависимостей и сборка фронтенда.
1. Настройка конфигурации (agents.yaml).
1. Установка сервисов (systemd для Linux, служба Windows).
1. Запуск.

**GUI:** минималистичный дизайн с шагами (Stepper), логом выполнения и индикатором прогресса.

## 4. Нефункциональные требования

- **Производительность** — интерфейс должен оставаться отзывчивым даже при длительной работе агентов (асинхронные запросы, WebSocket для логов).
- **Безопасность** — все API-ключи хранить в зашифрованном виде (например, в `.env` или системном хранилище). Пользователь `ai-agent` с ограниченными правами.
- **Масштабируемость** — возможность запускать несколько экземпляров бэкенда для параллельной обработки задач.
- **Локализация** — интерфейс на русском языке (перевод всех текстов).

## 5. Условия для разработки в России

- Все скрипты и установщик должны проверять доступность сервисов и предлагать альтернативные зеркала при недоступности.
- В установщике предусмотреть возможность ввода прокси перед началом загрузок.
- Документация должна содержать инструкции по настройке VPN и использованию прокси.
